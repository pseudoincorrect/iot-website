You may have noticed the huge title up there with this catchy acronyme : IoT . Well, what is it then ?

The Internet of things (IoT) is the extension of Internet connectivity into physical devices and everyday objects. Embedded with electronics, Internet connectivity, and other forms of hardware (such as sensors), these devices can communicate and interact with others over the Internet, and they can be remotely monitored and controlled

Wikipedia, Internet of things
TLDR; IoT represents technologies behind [electronic] devices connected to internet.

Few examples:

  • Wristwaches monitoring with data stored online
  • Remote controlled lamps, Hi-Fi, thermostats
  • Connected sensors in factories for maintenance applications
  • Online Package tracking
  • Smart cars
  • Sensors on power grid for energy distribution overview
  • Many mixes of physical devices and web applications
  • ...

It happens that I have been working in this domain mixing hardware, software and web technologies since 2014.

Most of my projects were focused around creating a custom physical device and connect it either to a local computer/smartphone or to a cloud application for data processing/presenting.

A typical application involves designing and implementing:

  • One or more Electronic Device(s)
  • Its/theirs Firmware
  • A way to Connect it to Internet
  • A Web Application (Fullstack)

Naturally, a large variety of tools and programming languages are involved in this process. I try, as much as I can, to pick the right tool for the current job.

I have mainly used:

  • C for firmware programming
  • Altium (or Eagle) for the Printed Circuit Board design
  • Python for scripting, DSP prototyping, algorithms design
  • Dart/Flutter for mobile application development
  • Python/Javascript for the backend development
  • Javascript/HTML5/CSS3 on the frontend
  • AWS for the deployment and CI/CD
  • Verilog when FPGAs are involved

Also a long list of Framework, SDK, Vendors IPs, vendor hardware are used along the way.
